# 图片过期时间功能

## 功能概述

EasyImages 2.0 支持为上传的图片设置过期时间，到期后会自动清理过期图片，节省存储空间。

## 功能特性

- **多种过期选项**：1天、1周、1个月、自定义时间、永久保存
- **自定义截至时间**：支持精确到分钟的自定义过期时间
- **自动清理**：定时清理过期图片
- **管理员控制**：可在后台开启/关闭功能
- **日志记录**：记录每张图片的过期时间
- **安全删除**：只删除过期图片，不影响其他文件

## 配置方法

### 1. 启用功能

1. 登录管理员后台
2. 进入 `图床设置` -> `上传设置`
3. 找到 `图片过期功能` 相关设置：
   - 启用 `图片过期功能`
   - 设置 `默认过期时间`
   - 启用 `自动清理`
   - 设置 `每日清理时间`

### 2. 设置定时清理

在服务器上添加 cron 任务，每天在指定时间运行清理脚本：

```bash
# 每天凌晨2点执行清理（根据配置的清理时间调整）
0 2 * * * /usr/bin/php /path/to/EasyImages/cleanup_expired_images.php >> /path/to/cleanup.log 2>&1
```

例如，如果设置的清理时间是凌晨2点，则添加：
```bash
0 2 * * * /usr/bin/php /home/www/EasyImages/cleanup_expired_images.php >> /home/www/cleanup.log 2>&1
```

### 3. 上传时选择过期时间

启用功能后，上传页面会显示过期时间选择器：
- **预设选项**：1天、1周、1个月、永久保存
- **自定义时间**：选择"自定义时间"后会显示日期时间选择器，可精确设置到期时间
- **时间验证**：自定义时间必须晚于当前时间
- 如果未选择，将使用默认设置

## 自定义时间使用说明

1. 在过期时间下拉框中选择"自定义时间"
2. 系统会显示日期时间选择器
3. 选择具体的到期日期和时间（最小时间为当前时间）
4. 上传时会按照设置的时间自动过期

## 技术实现

### 文件结构
- `cleanup_expired_images.php` - 清理脚本
- `app/function.php` - 日志记录函数更新
- `app/upload.php` - 上传处理更新
- `index.php` - 前端界面更新
- `admin/admin.inc.php` - 后台设置更新

### 数据库/日志
图片信息存储在 `admin/logs/upload/YYYY-MM.php` 文件中，包含：
- 过期时间选项 (`expiration`)
- 过期时间戳 (`expire_time`)

### 清理逻辑
1. 扫描所有上传日志文件
2. 检查每张图片的过期时间
3. 删除过期图片文件
4. 从日志中移除过期记录
5. 生成清理报告

## 注意事项

1. **备份重要数据**：启用自动清理前，请确保有备份
2. **权限设置**：确保 PHP 有删除文件的权限
3. **日志文件**：不要手动删除日志文件，否则可能导致清理异常
4. **时间同步**：确保服务器时间准确
5. **大文件处理**：清理大量文件时可能需要较长时间

## 故障排除

### 清理脚本不执行
- 检查 PHP 路径是否正确
- 检查文件权限
- 查看清理日志

### 图片未被清理
- 检查过期时间设置
- 确认 cron 任务是否正常运行
- 查看服务器时区设置

### 前端不显示过期选项
- 确认功能已启用
- 清除浏览器缓存
- 检查配置文件
